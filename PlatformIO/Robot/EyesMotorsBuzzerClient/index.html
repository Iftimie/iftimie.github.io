Robot - Eyes, Motors, Buzzer Client

A remote-controlled robot concept combining ESP32-CAM, motors, LED-matrix eyes, and a buzzer. This is the first full client build, with iterative hardware revisions and a custom PCB.

Story

I wanted to build a compact robot inspired by WALL-E: ESP32-CAM for vision, motors for movement, LED matrix eyes for expression, and a buzzer for sounds. The first version worked, but it had a lot of cables and everything was crammed into the small head. One bad touch could short something.

To clean up the wiring, I learned EasyEDA and designed my first PCB for the robot. The board was mostly header pins so I could plug components without soldering. When the PCBs arrived, I still had to solder the headers and fix a footprint mistake with a couple of jumper wires, but the board worked.

Hardware revisions

  • First version: fully wired build, hard to manage, high risk of shorts.
  • PCB v1: EasyEDA layout with header pins to simplify assembly.
  • Post-assembly: small rework due to footprint mismatch, but functional.

Software architecture

  • FreeRTOS tasks split responsibilities: WebSocket, Eyes, Audio, Motors.
  • WebSocketTask connects to the control server and parses commands or grouped JSON.
  • EyesTask runs LED matrix emotions, text, and sequences via a dedicated adapter.
  • MotorsTask sends movement commands over UART to an ATtiny motor driver.
  • Command queues coordinate actions and report done events for sequencing.

Key files: src/main.cpp, src/tasks/WebSocketTask.cpp, src/tasks/EyesTask.cpp, src/tasks/MotorsTask.cpp.

Related projects

  • WebServer: FastAPI broadcaster that sends command strings to WebSocket clients.
